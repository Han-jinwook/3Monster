# 🎨 Monster UI/UX 및 창 배치 규정 (UI/UX Guidelines)

- **문서 번호**: 3M-DOC-003
- **버전**: v1.1
- **갱신 일시**: 2026-06-04
- **관리 주체**: Monster 총괄 AI (Hub AI)

---

본 문서는 3Monster 프로젝트 계열의 모든 하부 프로그램 및 대시보드에 적용되는 공통 UI/UX 설계와 창 띄우기 배치 표준 규정입니다. 모든 프로그램 개발 시 본 규정을 근거로 화면을 구성하십시오.

## 1. 프로그램 창 띄우기 위치 규정 (Window Spawning Position)

모든 프로그램 실행 및 모달 팝업 시, 화면 노출 위치를 다음과 같이 일관되게 고정합니다.

1. **최초 진입 및 작은 모달/알림창 (Initial Modal / Popup)**
   - **가운데 정렬**: 라이선스 인증 창, 로그인 알림창, 에러 메시지 팝업 등 최초에 뜨는 작은 창들은 화면 **정중앙(Center)**에 배치합니다.
2. **메인 프로그램 실행 창 (Main Program Window)**
   - **좌상단 오프셋 배치**: 로그인 통과 후 실행되는 메인 프로그램 창의 좌상단(Top-Left) 시작점은 모니터의 실제 화면 좌상단 끝에서 **사방으로 약 1cm (화면 해상도 기준 약 35~45px) 가량 떨어진 지점**에 위치시킵니다.
   - **오프셋 예시**:
     - `x = 40px`, `y = 40px` 위치에 좌상단 끝을 맞춤.
     - 모니터 모서리에 화면이 완전히 달라붙어 윈도우 타이틀바나 테두리가 조작하기 불편해지는 현상을 방지합니다.

## 2. 파일 저장 및 윈도우 탐색기 연동 규정 (Data Saving & Explorer Link)

1. **다운로드 경로의 일원화**
   - 모든 수집 데이터 저장 및 다운로드 파일 출력은 사용자 시스템의 기본 `다운로드(Downloads)` 폴더 경로로만 수행합니다.
   - 프로그램 설치 경로 하위에 로컬 데이터 저장이나 엑셀 내보내기를 분산하여 저장하게 하거나, 유저에게 "기존 데이터 덮어쓰기"의 불안감 및 "DB 초기화" 작업을 강제하는 설계를 전면 금지합니다.
2. **파일명 명명 규칙 표준**
   - **포맷**: `[앱별 의미 있는 식별자].[확장자]`
   - **타임스탬프 제외**: 생성 일자는 이미 운영체제의 파일 속성(메타데이터)으로 표현되므로, 파일 이름 끝에 불필요한 일시(MMDD_HHMM 등)를 덧붙이는 규정은 전면 제외합니다.
   - **앱별 식별자 설계**:
     - 네이버 플레이스 수집기 (NPlace-DB): `[지역]_[수집키워드].[확장자]` (예: `서울 강남구_피부샵.csv`)
     - 네이버 카페 크롤러 (CafeCrawler): `[카페명]_[수집기간].[확장자]` (예: `카페수집_게시글_20260615_20260617.csv`)
3. **직접 저장(Direct Saving) 원칙**
   - 프로그램 내부의 임시 SQLite DB 파일 생성 후 사용자가 매번 수동 다운로드받는 다단계 구조를 배제합니다.
   - 애초에 수집 시작 시점 혹은 수집 진행 중에 사용자 `다운로드(Downloads)` 폴더 내 지정된 포맷의 CSV/Excel 파일에 실시간 혹은 수집 완료 즉시 **직접 저장(Direct Writing/Appending)** 되도록 설계합니다.
   - 수집이 끝나면 자동으로 저장 폴더(다운로드)의 해당 파일이 포커싱된 윈도우 탐색기가 실행되도록 구현합니다.
4. **불필요한 조작 버튼 삭제**
   - 타임스탬프 기반 고유 파일명 명명을 통해 덮어쓰기 위험이 사라지므로, 기존의 불필요한 "DB 초기화" 또는 "데이터 삭제" 버튼 및 관련 경고 알림창은 UI에서 완전히 제거하여 사용자 불안을 해소합니다.

## 3. 화면 디자인 레이아웃 공통 표준 (Layout Standards)

1. **로그 박스 모니터링 (Log Monitor)**
   - 실시간 로그 모니터는 줄 제한 없이 볼 수 있는 **무제한 스크롤 박스**로 구현하며, 문제 발생 시 빠른 대처를 위해 **원클릭 복사** 및 **고객센터 이동** 단축 버튼을 필수 탑재합니다.
2. **고객지원/고객센터 스레드 레이아웃**
   - 고객지원 창은 여백을 최소화한 슬림한 카드 형태로 공간을 최적화합니다.
   - 질문 본문의 추가 정보가 없는 경우, 답변 구분선 및 상단 여백을 없애 스레드를 제목에 밀착시킵니다.
   - 추가 문의 남발을 방지하기 위해, 관리자 답변이 미등록된 대기 상태에서는 사용자 추가 문의(댓글) 작성을 비활성화 처리합니다.
3. **프로필 설정 및 마이페이지 (My Page)**
   - 사용자 인증 완료 후 헤더의 이메일 ID를 클릭하여 진입합니다.
   - **좌측 영역**: 구매 리스트 (시리얼 키, 이용 기간, 라이선스 상태) 시각화.
   - **우측 영역**: 크몽 닉네임 수정(단독 저장 버튼 배치 - 수정 시에만 활성화) 및 알림 수신 토글 설정.
   - **로그아웃 버튼**: 마이페이지 우측 하단에만 깔끔한 Outline 스타일로 단독 배치하여 사용자의 우발적인 로그아웃을 방지합니다.

## 4. 설정 및 라이선스 로컬 저장소 표준 규정 (Local Settings & License Storage)

1. **라이선스 및 동기화 범위 최소화**
   - 3Monster 프로그램은 **1컴퓨터 1라이선스 원칙**으로 구동되므로, 다중 PC 간 설정 동기화나 클라우드 연동 등 추가 비용이 발생하는 고급 동기화 서비스는 개발 범위에서 제외합니다.
2. **투트랙 저장 표준 경로 규격화**
   - 프로그램 신규 버전 업데이트 혹은 실행 파일(EXE)의 폴더 위치 이동 시에도 사용자의 정보가 유실되거나 재인증을 요구하지 않도록 다음 두 가지 경로 규격을 유일한 표준으로 지정합니다.
     - **트랙 1: 수집 결과 데이터 및 텍스트 로그**
       - **Windows 기준 경로**: `%USERPROFILE%\Documents\MarketingMonster\[PRODUCT_ID]\` (내 문서 하위)
       - **저장 대상**: `database.sqlite` (로컬 DB), `Logs/` (수집/작업 로그)
     - **트랙 2: 라이선스 및 유저 설정 정보 (AppData 격리)**
       - **Windows 기준 경로**: `%APPDATA%\MarketingMonster\[PRODUCT_ID]\` (또는 `%APPDATA%\[PRODUCT_ID]\`)
       - **저장 대상**: `license.dat` (라이선스 인증키), `user_settings.json` (사용자 세팅), `templates.json` (메시지 템플릿)
   - 모든 3Monster 제품군은 이 투트랙 규칙을 준수하여 구현되어야 합니다.
