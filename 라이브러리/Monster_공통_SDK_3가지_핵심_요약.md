# 🔱 3Monster 공통 SDK 3가지 핵심 요약

- **문서 번호**: 3M-DOC-005
- **버전**: v1.0
- **갱신 일시**: 2026-05-19
- **관리 주체**: Monster 총괄 AI (Hub AI)

---

본 요약 문서는 3Monster 패밀리 개별 앱(NPlace-DB, 카페크롤러, 자동댓글러 등)에 공통 이식되는 **3대 핵심 SDK(인증, 업데이트, 로깅)**의 실체적 기능과 연동 방식, 그리고 허브앱과의 상호 유기적 관계를 한눈에 파악하기 위해 작성된 공식 요약집입니다.

---

## 🔒 1. 인증 엔진 (`auth.py`) - 라이선스 & HWID 기기 락킹

개별 앱의 보안 강화를 위한 핵심 모듈로, 1PC 1Key 원칙 및 실시간 라이선스 권한을 제어합니다.

### 💡 핵심 기능
*   **실시간 클라우드 검증**: 사용자가 입력한 라이선스 시리얼 키를 Supabase REST API를 통해 실시간으로 즉시 대조합니다. (외부 라이브러리 의존성이 없으며 **0.5초 타임아웃** 강제로 화면 멈춤 현상이 없습니다.)
*   **HWID 기기 강제 바인딩**: 최초 등록 시 사용자 PC의 고유 하드웨어 식별자(HWID)를 원격 서버에 영구 바인딩하여, 하나의 라이선스로 여러 대의 PC에서 부정 사용하는 행위를 원천 차단합니다.
*   **만료일 및 상태 실시간 추적**: 사용 기간 만료 또는 관리자 원격 차단(Block) 상태를 실시간으로 추적하여 권한이 없는 사용자의 진입을 차단합니다.

### 🖥️ UI/UX 작동 방식
*   기존 개별 앱 고유의 미려한 인증 UI 팝업 창 구조를 그대로 유지하며 백그라운드로 작동하여, 유저가 체감하는 UI 어색함이 전혀 없습니다.

## 🔄 2. 업데이트 엔진 (`updater.py`) - 원격 버전 감지 및 자동 패치

프로그램의 지속적인 보완 및 최신 기능 제공을 위해 클라우드 배포판과 연동되는 엔진입니다.

### 💡 핵심 기능
*   **최신 버전 실시간 비교**: 앱 구동 시 클라우드 데이터베이스(`app_versions`)를 조회하여 현재 로컬 버전과 서버 버전의 우위를 초고속으로 판별합니다.
*   **스트리밍 방식 대용량 다운로드**: 신규 패치 파일 감지 시 원격 다운로드 링크를 통해 끊김 없이 대용량 압축 패키지를 로컬로 스트리밍 다운로드합니다.
*   **자가 패칭 및 재시작 연동**: 다운로드가 완료되면 프로세스를 종료하고 기존 앱 고유의 배치 파일 헬퍼와 연동하여 자동으로 구버전을 신버전으로 교체한 뒤 앱을 즉시 재시작합니다.

### 🖥️ UI/UX 작동 방식
*   부팅 시 백그라운드로 작동하다가 업데이트 발견 시에만 `"새로운 업데이트가 있습니다. 지금 패치하시겠습니까?"` 모달 대화상자를 노출하여 유저의 행동 흐름을 방해하지 않습니다.

## 📝 3. 로깅 엔진 (`logger.py`) - 5MB 롤링 로그 및 원클릭 복사 헬퍼

앱 실행 중의 안정성을 도모하고, 에러 발생 시 유저가 허브앱 고객센터로 가장 편리하게 접수할 수 있도록 돕는 프리미엄 UX 지원 엔진입니다.

### 💡 핵심 기능
*   **5MB 용량 제한 Rotating Logging**: 에러 로그가 무한히 쌓여 유저 컴퓨터 하드디스크 용량을 과점유하는 문제를 방지하기 위해 단일 파일 **최대 5MB 크기 제한** 및 롤링 핸들러(최대 3개 백업)를 강제 적용합니다.
*   **디버그 보고서 규격화**: 오류 발생 시 HWID, 프로그램 식별자, 발생 일시, 상세 Traceback을 규격 리포트 템플릿 형태로 즉시 포맷팅합니다.
*   **무의존성 클립보드 자동 복사**: 별도의 외부 패키지 설치 없이 파이썬 내장 Tkinter 클립보드 API를 활용하여, 에러 검출 즉시 해당 디버그 리포트를 **유저 클립보드에 조용히 백그라운드 자동 복사**합니다.

### 🖥️ UI/UX 작동 방식 (통합 허브 1:1 고객센터 연동 시나리오)
1.  **에러 발생 및 자동 복사**: 앱 구동 중 에러가 나면 팝업 메시지 출력과 동시에 상세 오류 리포트가 클립보드에 자동 탑재됩니다.
2.  **허브앱 고객센터 접속**: 대시보드의 `[고객센터 문의하기]` 버튼을 누르면 우리 **3Monster 통합 허브앱의 1:1 문의 페이지**로 주소가 열립니다.
3.  **이메일 인증 로그인**: 허브웹 포털의 철저한 보안 장치에 따라 사용자는 최초 이메일 로그인 인증을 거쳐야만 문의 접수창에 접근 가능합니다.
4.  **원클릭 붙여넣기 접수**: 로그인 성공 후 유저는 문의 본문에 바로 **`Ctrl + V`**를 눌러 디버그 리포트를 붙여넣고 증상과 스크린샷만 추가해 전송하면 완벽하고 깔끔한 에러 접수가 완료됩니다.
